# M0 — Repo Scaffold Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the M0 scaffold for the Telegram → OlympTrade Signal Copier — a runnable, lint-clean, type-checked, Docker-buildable Python package with no functional behavior (real code lands in M1+).

**Architecture:** Single `pyproject.toml` (PEP 621 + hatchling build backend) drives everything. uv is the build/dev tool, providing a `uv.lock` for reproducible installs. The package lives under `src/signal_copier/` (src layout) with a stub `__main__.py` that prints a "not implemented" message and exits 0. A Dockerfile using `python:3.13` (full) and a `railway.toml` define the deployable shape. A `.pre-commit-config.yaml` wires ruff format + check into git (no mypy on commit). No CI in M0. No runtime deps in M0 — they're staged per milestone (M1: pydantic-settings + loguru, M4: asyncpg, M5: telethon, M8: vendored olymptrade_ws).

**Tech Stack:** Python 3.13+, uv (build + lockfile), hatchling (build backend), pytest + pytest-asyncio, ruff (lint + format), mypy --strict (typecheck), pre-commit (git hooks), Docker (Railway deploy).

**Spec reference:** `docs/superpowers/specs/2026-06-19-m0-scaffold-design.md`

## How to use this plan

1. Work through tasks in order. Each task is self-contained but builds on the previous.
2. Every step has explicit commands with expected output.
3. After every code change, run the verification step before committing.
4. If a verification fails, STOP. Read the error. Fix the issue. Re-run. Do not proceed to the next step until the current step's verification passes.
5. Commit frequently — one commit per task is the minimum.

**Working directory:** all commands assume you are at the project root (`olymptrade/`).

**Platform notes:** uv commands are cross-platform. Use PowerShell syntax on Windows, bash on macOS/Linux. File paths use forward slashes (works in both shells).

**Plan-vs-spec divergence note:** The spec's V-2 says "no tests ran" (exit 5). This plan adds one TDD test for the stub `main()` in Task 3. After Task 3, V-2 becomes "1 test passed" (exit 0). The spec's V-5 expected "2 source files"; this plan adds a test file, so V-5 becomes "3 source files". These are the only divergences; everything else matches the spec exactly.

- [ ] **Step 2: Write the failing test**

Create the file `tests/test_main.py` with these exact contents:

```python
def test_main_prints_scaffold_message(capsys: Any) -> None:
```

Notes:
- `Any` for the `capsys` parameter sidesteps the need for `pytest` type stubs (pytest 8 doesn't ship `.pyi` files). mypy --strict accepts `Any` in a parameter position.
- The test asserts only on a stable substring of the expected message, so the assertion is robust to minor wording tweaks.
- `signal_copier.__main__` does not exist yet — the import will fail. This is the "red" step of red-green-refactor.

- [ ] **Step 3: Run the test to verify it fails**

Run:
```bash
uv run pytest tests/test_main.py -v
```

Expected: FAIL with `ModuleNotFoundError: No module named 'signal_copier'`. (The package doesn't exist yet — that's the point.) The exit code is non-zero (likely 2 or 3).

If you see anything other than `ModuleNotFoundError`, stop and check the test file for typos.

- [ ] **Step 4: Implement the stub `main()` (the "green" step)**

Create the file `src/signal_copier/__main__.py` with these exact contents:


Notes:

- `print(...)` writes to stdout. The exact string must match the spec's V-6 expected output.
- The `if __name__ == "__main__":` block makes `python -m signal_copier` invoke `main()`.

## Task 7: Create Dockerfile

**Files:**
- Create: `Dockerfile`

- [ ] **Step 1: Write `Dockerfile`**

Create the file `Dockerfile` (no extension) at the project root with these exact contents:

```dockerfile
FROM python:3.13

WORKDIR /app

# Install uv
COPY --from=ghcr.io/astral-sh/uv:latest /uv /uvx /usr/local/bin/

# First layer: copy only what's needed for `uv sync` (cache-friendly)
COPY pyproject.toml uv.lock ./
COPY src/signal_copier/__init__.py ./src/signal_copier/__init__.py
COPY src/signal_copier/__main__.py ./src/signal_copier/__main__.py

# Install runtime deps only (--no-dev skips pytest/ruff/mypy in the image)
RUN uv sync --frozen --no-dev

# Now copy the rest
COPY src/ ./src/
COPY migrations/ ./migrations/

# Run as non-root
RUN useradd --create-home --shell /bin/bash app && chown -R app:app /app
USER app

CMD ["python", "-m", "signal_copier"]
```

Notes:
- `FROM python:3.13` is the full Debian image (~900 MB) per the spec's D-6 decision. Larger than `slim` but no musl/openssl compat issues.
- The first `COPY` step copies only `pyproject.toml`, `uv.lock`, and the two required package files. This is the Docker build cache layer — if you change a non-package file (e.g., add a test), the layer is reused and only the changed files are re-copied.
- `__main__.py` is in the first layer because hatchling requires at least one non-`__init__.py` Python file in a package to build a wheel. If you remove it, `uv sync --frozen --no-dev` will fail with a build error.
- `RUN uv sync --frozen --no-dev` — `--frozen` refuses to modify `uv.lock` during build (catches "I forgot to run `uv lock`" mistakes). `--no-dev` skips pytest/ruff/mypy in the image.
- `USER app` runs the container as a non-root user. Required for Railway best practice.
- No `LICENSE` file is `COPY`-ed (per D-2: Proprietary, no LICENSE).
- No `COPY` of `src/olymptrade_ws/` directly — it gets pulled in by `COPY src/ ./src/` in the second step, since it's under `src/`.

- [ ] **Step 2: Verify the Dockerfile has no obvious syntax issues**

Run:
```bash
docker build --check -t signal-copier-m0 . 2>&1 | head -20
```

Note: `docker build --check` is a Docker 27+ feature for syntax validation. If your Docker doesn't support it, skip this step — the full build in T12 will catch any issues.

Expected: either "OK" or "syntax OK" (no errors), OR a "flag not recognized" message (in which case skip — not a failure).

If your Docker supports it and you see errors, fix the Dockerfile and re-run.

- [ ] **Step 3: Commit**

```bash
git add Dockerfile
git commit -m "Add Dockerfile (python:3.13 full, uv-based install, M0)"
```
