# Vendored: `olymptrade_ws`

This directory contains a **vendored copy** of a third-party library. It is not generated by this project, not installed as a Python package, and not built from source at any point.

## Upstream

- **Source:** [`OlympTradeAPI`](https://github.com/) by **Chipa** (2025)
- **License:** MIT — see [`LICENSE`](./LICENSE) in this directory
- **Upstream contents copied:** the `olymptrade_ws/` subpackage (9 Python files, 1 markdown log)
- **Excluded from vendoring:** the rest of the upstream repository (root-level `simple_bot.py`, `test.py`, `test_api_methods.py`, `REFERENCE.md`, `.github/`, root `README.md`, root `LICENSE`). Only the importable `olymptrade_ws/` package is needed by this project.

## Why vendored (not installed as a package)

1. Upstream has **no `pyproject.toml` / `setup.py`** — it is source code, not a distributable package.
2. The project needs to **patch the code in-place** as the broker protocol evolves (reverse-engineered, breaks often). Vendoring = local modifications live here, no PR round-trip upstream.
3. **Deployment simplicity:** one `COPY . .` in the Dockerfile, no build-time git network calls, no submodule init, no PyPI pinning for an unpublished package.

## Import contract

- The package name is preserved as `olymptrade_ws`, so internal absolute imports inside the vendored code (e.g. `from olymptrade_ws.olympconfig import parameters`) continue to work **without modification**.
- From project code, import as: `from olymptrade_ws import OlympTradeClient` (or `BalanceAPI`, `MarketAPI`, `TradeAPI` — see `olymptrade_ws/__init__.py` for the full export list).

## Local modifications

- 2026-06-25: `core/connection.py` — renamed `extra_headers=` to `additional_headers=` in the `websockets.connect()` call. Upstream code targets an older `websockets` version; our project pins `websockets>=16.0` where the kwarg was renamed. Without this patch, OlympTradeBroker fails at startup with `BaseEventLoop.create_connection() got an unexpected keyword argument 'extra_headers'`.

## Re-vendoring

To refresh this directory from upstream:

```bash
# from the project root, assuming the upstream clone is at ../OlympTradeAPI
rm -rf src/olymptrade_ws/api src/olymptrade_ws/core src/olymptrade_ws/olympconfig
rm    src/olymptrade_ws/__init__.py src/olymptrade_ws/main.py
cp -r ../OlympTradeAPI/olymptrade_ws/* src/olymptrade_ws/
cp    ../OlympTradeAPI/LICENSE       src/olymptrade_ws/LICENSE
```

Then review the diff. If upstream changed internal imports or public API, update the call sites in `signal_copier/broker/olymp.py` and any other consumer, and record the change under "Local modifications" above.
